# The Sovereign Platform: Bulletproof Architecture (v5.0)

## 1. File Structure Topology
The zero-code restriction demands a flat, heavily modular static structure that relies purely on client-side JS executing real-time Firestore listeners.

```
/TradeSovereign
├── index.html                 # 100% Static Neubrutalist Landing
├── admin.html                 # Complete SPA Control Center
├── /student-zone
│   ├── index.html             # Curriculum Matrix + Gamification Edge
│   └── chapter.html           # D2L Lumi AI Tutor Interface
├── /trading-zone
│   └── index.html             # TradeOS Terminal & Workflow Builder
├── /functions                 # Serverless Edge Proxies
│   ├── api-proxy.js           # Escrows OpenAI & Polygon Keys
│   └── admin-auth.js          # MFA Token Verifier
├── /report                    # The Library of Alexandria
│   ├── ARCHITECTURE.md
│   ├── RISK_MITIGATION.md
│   ├── TIMELINE.md
│   └── [All standard guides...]
```

## 2. Firestore Document Modeling & Collections
The entire ecosystem relies on NoSQL document listeners (`onSnapshot`) to reflect instant changes from `/admin.html`.

- **`/classes/{classId}`**: Stores Class Names, Icon URLs.
  - **`/classes/{classId}/subjects/{subjectId}`**: Stores Subjects.
    - **`/classes/{classId}/subjects/{subjectId}/chapters/{chapterId}`**: Stores video URIs, AI Quiz JSON matrices.
- **`/admin/settings`**: 
  - `apiKeys`: `{"polygon": "encrypted_blob", "openai": "encrypted_blob"}`
  - `featureToggles`: `{"lumi_tutor": true, "chaplin_bot": false}`
- **`/logs`**:
  - `errors`: Stores `window.onerror` stack traces and user-agents.
  - `audit_trail`: Admin action tracking (Who rotated keys, who disabled features).
- **`/ai_recommendations`**: Auto-generated by cron-job Edge Functions parsing the `/logs`.

## 3. Serverless API Proxy Layer
To guarantee zero exposed keys, `admin.html` writes encrypted key strings to `/admin/settings/apiKeys`. 
When the `student-zone` requests an OpenAI chat completion, it POSTs to `https://us-central1-project.cloudfunctions.net/api-proxy`. The Edge Function verifies the Firebase Auth token, reads the physical key from Firestore, fires the request to OpenAI, and returns the JSON payload safely to the DOM.

## 4. UI Component & State Management Approach
- **UI Paradigm**: Neubrutalism + Liquid Glass. Using raw Tailwind CDN overridden with complex CSS Variables for `box-shadow` and `backdrop-filter`.
- **State**: `localStorage` acts as a synchronous bridge for theme variables (Dark/Light) and immediate Gamification streaks, while Firebase `onSnapshot` handles authoritative asynchronous DOM injections (disappearing chat widgets when the admin toggles them off).

## 5. Security Rules Execution
```javascript
match /admin/{document=**} {
  allow read, write: if request.auth.token.admin == true && request.auth.token.mfa_verified == true;
}
match /logs/{document=**} {
  allow write: if true; // Client pushing errors
  allow read: if request.auth.token.admin == true;
}
```
